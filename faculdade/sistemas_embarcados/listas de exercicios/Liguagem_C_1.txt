2.a) 
	O modificador "volatile" é utilizado na hora de declarar variaveis e é muito importante para sistemas embarcados que
	precisam lidar com o acesso à registradores. Esse modificador indica ao compilador que o valor da variável em questão
	deve ser recarregado toda vez que ela for solicitada. Isso faz sentido em sistemas embarcados pois geralmente os valo-
	res de variáveis são provenientes de registradores que podem ter seu valor alterado de forma não explícita por fatores
	externos.

3.c)
	if (REG & 0b00000100) {
		// codigo
	}
	else {
		// outro codigo
	}

	Esse bloco de código funciona porque, caso o bit 2 esteja setado, o resultado é maior que 0, que é interpretado como True
	em C. Enquanto que caso o bit 2 não esteja setado o resultado é 0, que é interpretado como False em C.

4.a)
	Um ponteiro em C é uma variável que, ao invés de armazenar um valor, armazena uma posição na memória. Eles são importantes
	em sistemas embarcados pois os periféricos físicos do microcontrolador são mapeados em posições de memória, então a unica
	forma de acessar os registradores responsáveis pelos periféricos é se utilizando de ponteiros.

	Unions são uma estrutura de dados em C que se caracterizam por guardar multiplos valores
	no mesmo espaço de memória de maneira mutualmente exclusiva. Os unions tem o tamanho do

7.a)
	Ambas as ferramentas são usadas para definir valores imutáveis mas divergem na forma como
	fazem isso. O #define funciona com o pré-processador substituindo todos os locais onde a
	variável aparece pelo valor literal definido, economizando memória pois não precisou-se sal-
	var uma variável na memória. Enquanto isso, o const define um valor apensa de leitura, onde 
	o compilador se encarrega de garantir que o valor não será manipulado de forma alguma.

7.b)

8.a)
	Interrupções são um mecanismo computacional usado para pausar o processamento atual da CPU
	para executar um bloco de código mais importante no momento. Em C, as interrupções são tra-
	tadas por funções especiais chamadas de ISR ( Interrupt Service Routines ), que são chamadas
	exclusivamente pelo próprio hardware.

8.b)

	}

10.a)
	Algumas práticas para a otimização de código em C são: 
	1 - Evitar o uso do tipo "int", preferindo os tipos do <stdint.h> para economizar memória.
	2 - Usar o prefixo "const" para gravar valores na ROM e economizar RAM.
	3 - Utilizar operações bit a bit pois são bem mais rápidas.
	4 - Utilizar as flags de otimização do compilador.
	
10.b)
	int multiply_by_seven(int num) {
		return num * 7;
	}
	
	#include <stdint.h>
	uint16_t multiply_by_seven(const uint8_t num) {
		return (num << 3) - num;
	} 
	
	obs: (num << 3) = num * 8, porém utilizando operações bit a bit. logo, subtrair num dessa 
			operação equivale a multiplicar por 7 mas de maneira muito mais rápida.
